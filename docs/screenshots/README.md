# Screenshots

This folder is reserved for real screenshots captured from the running healthcare care coordination portal. Do not add fake or generated screenshots.

## Recommended Captures

| Screen | Suggested file | Notes |
|---|---|---|
| Dashboard | `dashboard.png` | Show portfolio command center metrics and operational summary. |
| Patient list | `patient-list.png` | Show synthetic patient data only. |
| Appointment scheduling | `appointment-scheduling.png` | Show synthetic patient/provider selection and scheduling fields. |
| Care plan details | `care-plan-details.png` | Show goals, instructions, and follow-up tasks. |
| Clinical insight analysis | `clinical-insights.png` | Include the responsible AI notice and synthetic note only. |
| Follow-up tasks | `follow-up-tasks.png` | Show task status, priority, and due-date workflow. |
| Notification simulation | `notification-simulation.png` | Show simulated delivery status, not real notification delivery. |
| Audit log | `audit-log.png` | Show safe audit metadata and correlation ID traceability. |
| System health | `system-health.png` | Show API readiness and health status. |
| Azure deployment diagram | `azure-deployment.png` | Optional exported diagram from `architecture/diagrams/azure-deployment.md`. |

## Capture Checklist

- Use synthetic demo data only.
- Do not show local secrets, `.env` files, keys, tokens, connection strings, or terminal history containing secrets.
- Do not show real patient, provider, hospital, employer, client, or clinical data.
- Keep browser zoom and viewport consistent across screenshots.
- Prefer a clean desktop browser window with the portal filling the frame.
- Review each image before committing it.
