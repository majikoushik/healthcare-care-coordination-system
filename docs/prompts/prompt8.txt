Epic 7 Goal:

Allow users to create, view, update, and track follow-up tasks connected to care plans, patients, providers, and clinical insight recommendations.

Follow-up Task Tracking should demonstrate healthcare coordination workflow, task ownership, priority management, due-date tracking, overdue detection, and care-plan execution visibility.

This epic should build on the existing Care Plan and Clinical Insights modules.

Validation rules:

* Care Plan ID is required.
* Title is required.
* Title should have a reasonable length limit.
* Description is required.
* Description should have a reasonable length limit.
* Task type must be a supported value.
* Priority must be a supported value.
* Status must be a supported value.
* Due date is required.
* Due date should not be in the past for new pending tasks.
* Assigned to is required.
* Completion notes should have a reasonable length limit.
* Completed timestamp should be set when task status becomes Completed.
* Created timestamp should be system generated.
* Updated timestamp should be system generated or updated automatically.

If adding an optional action such as “Create task from clinical insight,” keep it simple and human-controlled.

Optional endpoint:

* `POST /api/v1/clinical-insights/{clinicalInsightId}/follow-up-tasks`

This endpoint should create a follow-up task from a reviewed clinical insight recommendation only if practical.

Important responsible AI rule:
