Epic 6 Goal:

Prepare the Clinical Note Insights module for Azure AI Language / Text Analytics for Health integration without requiring real Azure credentials for local development.

This epic should introduce a clean Azure AI provider implementation path, configuration model, feature toggle, documentation, and safe fallback behavior.

The application must continue to work locally using the existing mock AI provider.

Important:

Do not hardcode real Azure AI credentials.

Do not require a real Azure AI endpoint or key for local development.

Do not use real patient data or real clinical notes.

Do not present Azure AI output as medical advice or diagnosis.

Recommended provider modes:

* `Mock`
* `AzureAIReady`
* `AzureAIConfigured`

Suggested configuration section:

```json
{

    "ProviderMode": "Mock",
    "AzureLanguage": {
      "Endpoint": "",
      "Key": "",
      "Region": "",
      "ModelVersion": "latest"
    }
  }
}
```

Expected behavior:

1. If `ProviderMode` is `Mock`, use `MockClinicalTextAnalyzer`.
2. If `ProviderMode` is `AzureAIReady` but credentials are missing, return a clear configuration-readiness response or fallback safely to mock depending on existing architecture.
3. If `ProviderMode` is `AzureAIConfigured` and valid configuration is present, route through the Azure provider implementation path.
4. If the actual SDK integration is added, it must be isolated behind the provider abstraction.
5. If the actual SDK integration is not added yet, create a clean placeholder with documentation and tests explaining the future integration point.
