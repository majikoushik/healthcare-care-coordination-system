Allow users to schedule, view, update, and track synthetic healthcare appointments between patients and providers through backend APIs and React frontend.

Appointment Scheduling connects Patient and Provider workflows and demonstrates healthcare coordination, status lifecycle management, validation, traceability, and enterprise API design.

* Appointment ID
* Patient ID

* Visit reason

* Notes
* Created timestamp
* Updated timestamp

Supported appointment types:

* Consultation
* FollowUp
* LabReview
* MedicationReview
* CarePlanReview

Supported appointment statuses:

* Requested
* Scheduled
* CheckedIn
* Completed
* Cancelled
* NoShow

Validation rules:

* Patient ID is required.
* Patient ID must refer to an existing patient if patient lookup is available.
* Provider ID is required.
* Provider ID must refer to an existing provider if provider lookup is available.
* Appointment date and time is required.
* Appointment date and time should not be in the past for new appointments.
* Visit reason is required.
* Visit reason should have a reasonable length limit.
* Appointment type must be a supported value.
* Status must be a supported value.
* Notes should have a reasonable length limit.
* Created timestamp should be system generated.
* Updated timestamp should be system generated or updated automatically.

Recommended appointment status transition rules:

* Requested → Scheduled
* Requested → Cancelled
* Scheduled → CheckedIn
* Scheduled → Cancelled
* Scheduled → NoShow
* CheckedIn → Completed
* CheckedIn → Cancelled
* Completed → no further transition
* Cancelled → no further transition
* NoShow → no further transition

Suggested API endpoints:

* `POST /api/v1/appointments`
* `GET /api/v1/appointments`
* `GET /api/v1/appointments/{appointmentId}`
* `GET /api/v1/patients/{patientId}/appointments`
* `GET /api/v1/providers/{providerId}/appointments`
* `GET /api/v1/appointments/date/{date}`
* `PUT /api/v1/appointments/{appointmentId}`
* `PATCH /api/v1/appointments/{appointmentId}/status`

Suggested status update request:

```json
{
  "status": "Scheduled",
  "reason": "Provider confirmed the requested appointment slot",
  "updatedBy": "System"
}
```
