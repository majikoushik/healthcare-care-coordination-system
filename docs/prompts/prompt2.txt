Epic 1 Goal:

Allow users to register and view synthetic patient profiles through backend APIs and React frontend.

Patient data in this project must be synthetic only. This feature demonstrates secure healthcare data handling patterns, not a production medical record system.

Supported consent statuses:

* NotProvided
* Provided

Supported gender values for demo:

* Male
* Female
* Other
* PreferNotToSay

Validation rules:

* Full name is required.
* Full name should have a reasonable length limit.
* Date of birth is required.
* Patient must not have a future date of birth.
* Email is required and must be valid.
* Mobile number is required and must be valid enough for demo use.
* Address is required.
* Emergency contact name is required.
* Emergency contact number is required.
* Consent status must be a supported value.
* Created timestamp should be system generated.
* Updated timestamp should be system generated or updated automatically.

Healthcare privacy and compliance-readiness expectations:

This epic handles synthetic patient profile data, so safe healthcare data handling must be emphasized.

Ensure implementation and documentation state:

1. Only synthetic patient data is used.
2. This is not a production medical record system.
3. The project does not claim HIPAA compliance or medical certification.
4. Patient data should not be logged in full.
5. Logs should use safe metadata such as patient ID and correlation ID.
6. Sensitive health details should not be stored in patient profile metadata.
7. Future authentication and role-based access control should be documented.
8. Consent status is included as a compliance-readiness pattern, not a complete consent management system.
