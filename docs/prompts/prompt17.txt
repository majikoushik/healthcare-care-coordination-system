Run a Security, Secrets, Privacy, and Responsible AI Review for the `healthcare-care-coordination-system` repository.

This is not a new feature epic. This is a focused final safety and security review before publishing the repository on GitHub.

Before starting:

1. Read `AGENTS.md`.
2. Review the complete repository after Epic 0 through Epic 14 and the Final QA pass.
3. Review backend services, React frontend, configuration files, Docker files, GitHub Actions, Bicep files, documentation, samples, tests, and architecture diagrams.
4. Do not add unrelated business features.
5. Do not change the core architecture unless a security or privacy issue requires it.

Goal:

Ensure the repository is safe to publish publicly and does not expose secrets, real data, unsafe healthcare claims, misleading AI claims, or insecure configuration patterns.

Main review areas:

1. Secrets review
2. Healthcare privacy review
3. Responsible AI review
4. RBAC/security readiness review
5. Logging and audit safety review
6. Configuration safety review
7. DevOps and Azure deployment safety review

Search the full repository for any real or suspicious values:

* API keys

* Client secrets
* Tenant IDs

* Private keys
* Certificates
* Real email credentials

* `.ts`
* `.tsx`
* `.env`
* `.yml`
* `.yaml`
* `.bicep`
* `.md`

* `.csproj`
* Dockerfiles
* GitHub Actions workflows
* Bicep parameter files
* sample files
* test files

Expected result:

1. No real secrets exist.
2. `.env.example` contains placeholders only.
3. No `.env` file is committed.
4. Bicep parameters contain placeholders only.
5. GitHub Actions use GitHub Secrets placeholders only.
6. Docker Compose does not contain production secrets.
7. Documentation does not include real secret examples.

If any secret-like value exists:

1. Remove it.
2. Replace it with a safe placeholder.
3. Update `.gitignore` if needed.
4. Document the fix.
5. Do not invent fake values that look like real secrets.

* `clinically certified`
* `production medical system`
* `medical diagnosis`

* `privacy-aware design`

* `does not provide medical advice`

Responsible AI review checklist:

Review Clinical Insights and Azure AI Language readiness.

Ensure:

1. Mock AI provider is the default for local development.
2. Azure AI Language provider is optional and configuration-driven.
3. No Azure AI credentials are required locally.
4. No real clinical notes are used.
5. No real AI endpoint or key is committed.
6. AI output is clearly marked as assistive only.
7. AI output requires human review.
8. AI output is not presented as diagnosis.
9. AI output is not presented as medical advice.
10. AI output is not automatically converted into patient-facing communication.
11. AI-generated recommendations do not automatically become follow-up tasks without explicit human action.
12. Raw AI responses are not logged.
13. Full clinical note text is not logged.
14. Azure AI configuration is not exposed in the frontend.

Add or strengthen disclaimers where needed.
