Epic 11 Goal:

Strengthen the application’s observability, troubleshooting, health monitoring, logging, correlation, and production-readiness patterns.

This epic should demonstrate that the healthcare care coordination system is designed not only to work, but also to be monitored, diagnosed, supported, and operated in an enterprise cloud environment.

Recommended endpoints:

* `GET /health/live`
* `GET /health/ready`

Recommended response shape:

```json
{
  "service": "Patient.Api",
  "status": "Healthy",
  "environment": "Development",
  "correlationId": "correlation-id",
  "timestamp": "2026-07-15T10:00:00Z",
  "checks": [
    {
      "name": "self",
      "status": "Healthy"
    },
    {
      "name": "sql-server-readiness",
      "status": "Healthy"
    }
  ]
}
```

Do not expose secrets, connection strings, API keys, Azure AI endpoint details, Cosmos DB keys, SQL credentials, stack traces, or internal infrastructure secrets in health responses.

Logging expectations:

Use structured logging with safe metadata.

Logs may include:

* Correlation ID
* Request path
* HTTP method
* Status code
* Duration
* Service name
* Entity ID
* Event type
* Outcome
* Source service
* Provider mode, for AI module
* Role name, where safe
* Permission name, where safe

Logs must not include:

* Full patient profile
* Full clinical note text
* Full care plan clinical summary
* Full care plan instructions
* Full notification message content
* Full AI raw response
* Access tokens
* API keys
* Azure AI keys
* Cosmos DB keys
* SQL connection strings
* Client secrets
* Passwords
* Stack traces in API responses
