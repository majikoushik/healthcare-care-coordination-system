# Non-Functional Requirements

- Security: no committed secrets, future JWT/RBAC readiness, intentional CORS.
- Privacy: synthetic data only, safe logging, limited audit metadata.
- Availability: health checks and Azure Container Apps readiness.
- Observability: correlation IDs, structured logs, Application Insights readiness.
- Maintainability: domain-oriented service boundaries and shared building blocks.
- Testability: placeholder test projects from Epic 0, expanded per feature epic.
- Portability: Docker Compose local foundation and Azure deployment direction.
